# Этапы

## Регистрация и авторизация

Регистрация и вход — отдельные страницы (`/register`, `/login`).

При регистрации указываются:
- email
- пароль (минимум 8 символов)
- ФИО (обязательное)
- Должность (обязательное)

Каждому пользователю автоматически присваивается последовательный номер (`number`). Номер уникален и используется при формировании лотов производства.

Список зарегистрированных пользователей отображается на странице `Исполнители`.

## Заполнение словарей

### Позиции

Список позиций уникальных по полю `code`.

Позиция состоит:
- название — человекочитаемое значение
- код — уникальный код позиции
- тип
  - обычный — нет уникальной обработки
  - проверяемый — перед использованием необходимо пройти контроль качества
  - производимый — результат производственного этапа

### Поставщики

Поставщик состоит:
- название — человекочитаемое значение

## Карантин

Материалы поступают через карантинные счета на странице `Карантин`.

Счёт содержит:
- дата
- номер счёта
- лот (уникальный)
- поставщик
- список позиций с количеством

### Контроль качества

Для проверяемых позиций в счёте проводится контроль качества:
1. указывается принятое количество и количество брака
2. генерируется лот в формате `КОД-ДДММГГ-НОМЕР_СЧЁТА-N`, где N — порядковый номер КК для данной связки
3. если тот же пользователь повторно проводит КК по той же позиции/счёту/дате — количество добавляется к существующей поставке (N не инкрементируется)
4. если другой пользователь — создаётся новая поставка с инкрементом N
5. результат КК создаёт поставку на складе со статусом `Completed`

## Склад

На странице `Склад` отображаются все поставки (результат карантинного КК и производства).

## Резерв

Перед производством чипов материалы необходимо зарезервировать на странице `Резерв`:
1. указывается желаемое количество чипов
2. система автоматически резервирует необходимые материалы из доступных поставок
3. зарезервированные материалы списываются со склада (`usedQuantity` увеличивается)

Из резерва можно:
- **произвести** — указать количество произведённых чипов и/или брак
- **вернуть остаток** — неиспользованные материалы возвращаются на склад

## Производство

### Производство чипов (из резерва)

Производство чипов выполняется со страницы `Резерв`:
1. выбирается резерв
2. указывается количество произведённых чипов и/или испорченных материалов
3. результат ставится на склад с лотом формата `ДДММГГ-НОМЕР_ПОЛЬЗОВАТЕЛЯ-N`
4. уникальность лота определяется комбинацией `рецепт + дата + номер пользователя + лоты материалов`
5. в лог производства (`manufacturingProduction`) записывается id пользователя

### Упаковка и Отгрузка

Упаковка и Отгрузка представлены на страницах `Упаковка` и `Отгрузка`.
Используют общий компонент производства, различаются только рецептом.

1. перед производством проверяется наличие материалов, если какого-либо нет, то будет показана ошибка
2. перед производством проверяется возможное количество производства
3. производить за раз можно **только из одной комбинации лотов материалов**
4. в результате будет показана таблица с необходимыми материалами, их лотами и количеством
5. результат производства будет поставлен на склад, используемые позиции будут со склада списаны
6. результирующая поставка будет иметь лот — уникальная комбинация `состав + пользователь + дата производства`
