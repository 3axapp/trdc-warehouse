# Этапы

## Заполнение словарей

### Позиции

Список позиций уникальных по полю `code`.  
Позиция состоит:
- название - человекочитаемое значение
- код - уникальный код позиции
- тип
  - обычный - нет уникальной обработки
  - проверяемый - перед использованием необходимо пройти контроль качества
  - производимый - (пока нет уникальной обработки)

### Поставщики

Поставщик состоит:
- название - человекочитаемое значение

### Исполнители

_Пока отдельная коллекция, возможно потом будет из списка зарегистророванных пользователей с ролью `Исполнитель`._

Исполнитель состоит:
- название - человекочитаемое значение

## Поставка материалов

На странице `склад` можно добавить поставку, и на ней же, если это поставка проверяемой позиции, можно провести контроль качества.  

## Производство

Производство представлено на трёх страницах: `Производство`, `Упаковка`, `Отгрузка`.  
Различия только в используемом рецепте.  

1. перед производством проверяется наличие материалов, если какого-либо нет, то будет показана ошибка
2. перед производством проверяется возможно количество производства
3. производить за раз можно **только из одной комбинации лотов материалов**
4. в результате будет показана таблица с необходимыми материалами, их лотов и количества
5. результат производства будет поставлен на склад, используемые позиции будут со склада списаны
6. результирующая поставка будет иметь лот. Новый лот - уникальная комбинация `состав + исполнитель + дата производства`
